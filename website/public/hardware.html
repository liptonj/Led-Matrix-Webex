Build your LED Matrix Webex Display

Required Components

Component Description Notes
ESP32 or ESP32-S3 Microcontroller board ESP32-S3 recommended for better performance
HUB75 LED Matrix Panel 64x32 or 64x64 pixels RGB LED matrix display
5V Power Supply 3-5A recommended Depends on matrix size and brightness
Jumper Wires Female-to-female For connecting ESP32 to LED matrix

Wiring Diagram

ESP32-S3 to HUB75 Matrix

HUB75 Pin ESP32-S3 GPIO Function
R1GPIO 42Red data (upper half)
G1GPIO 41Green data (upper half)
B1GPIO 40Blue data (upper half)
R2GPIO 38Red data (lower half)
G2GPIO 39Green data (lower half)
B2GPIO 37Blue data (lower half)
AGPIO 45Row select A
BGPIO 48Row select B
CGPIO 47Row select C
DGPIO 21Row select D
EGPIO 14Row select E (64px height)
CLKGPIO 2Clock signal
LATGPIO 1Latch signal
OEGPIO 46Output enable

Note: ESP32 (non-S3) uses different GPIO pins. See the full documentation in the GitHub repository.

Assembly Steps

  1. Connect the ESP32 to the HUB75 matrix according to the wiring diagram above
  2. Connect the 5V power supply to the matrix power input
  3. Flash the bootstrap firmware to the ESP32 using esptool or PlatformIO
  4. Power on the device and connect to the WiFi AP "LED-Matrix-Setup"
  5. Follow the web-based setup wizard to configure WiFi and Webex

Flashing Firmware

Option 1: Using esptool (Recommended for beginners)

# Install esptool
pip install esptool

# Flash ESP32-S3 bootstrap firmware
esptool.py --chip esp32s3 --port /dev/ttyUSB0 write_flash 0x0 bootstrap-esp32s3.bin

# Flash ESP32 bootstrap firmware
esptool.py --chip esp32 --port /dev/ttyUSB0 write_flash 0x0 bootstrap-esp32.bin

Option 2: Using PlatformIO

# Clone the repository
git clone https://github.com/liptonj/Led-Matrix-Webex.git
cd Led-Matrix-Webex/firmware_bootstrap

# Upload to ESP32-S3
pio run -e esp32s3 -t upload

# Upload to ESP32
pio run -e esp32 -t upload

Troubleshooting

Display not working?

  • Check all wiring connections match the pinout table
  • Verify power supply provides sufficient current (3-5A)
  • Try reducing brightness in the web configuration

Can't connect to WiFi AP?

  • Reset the device by holding the reset button for 10 seconds
  • Check that "LED-Matrix-Setup" network appears in your WiFi list
  • Some devices require manual connection to captive portal at 192.168.4.1

Webex connection failing?

  • Verify your Webex integration client ID and secret are correct
  • Check that the device has internet access
  • Review logs in the web interface for specific error messages
